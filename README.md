
# **Техническое задание: Recipes App**

## 1. **Описание проекта**

Веб-приложение для хранения, просмотра и редактирования рецептов.
Каждый рецепт представлен карточкой, включающей название, фото, список ингредиентов, количество порций и итоговую стоимость.
Приложение позволяет масштабировать ингредиенты в зависимости от количества порций и автоматически пересчитывает стоимость.

---

## 2. **Основной функционал**

### 2.1. Рецепты

- Добавление нового рецепта.

- Удаление рецепта.

- Редактирование существующего рецепта (название, фото, описание, количество порций).

- Просмотр полной информации о рецепте.


### 2.2. Ингредиенты в рецепте

- Добавление и удаление ингредиентов внутри рецепта.

- Изменение названия ингредиента.

- Изменение количества (через кнопки +/– или вручную).-

- Изменение единиц измерения (гр/мл/шт)

- Автоматический пересчёт стоимости ингредиента (количество × цена).


### 2.3. Автоматический пересчёт порций

- Рецепт хранит количество порций по умолчанию (2 порции).

- Пользователь может устанавливать другое количество порций.

- При изменении порций автоматически пересчитывается:

    - количество каждого ингредиента;

    - стоимость каждого ингредиента;

    - итоговая стоимость рецепта.


### 2.4. Страница продуктов (goods)

- Просмотр списка всех продуктов.

- Редактирование продукта: название, смайлик/иконка, цена, единица измерения.

- Удаление продукта.

- Поиск по названию.

- Наличие кнопки _“Добавить продукт”_.


---

## 3. **Структура данных**

### 3.1. Карточка рецепта

- `id`

- `title` — название

- `imageUrl` — изображение

- `description` — краткое описание

- `servings` — базовое количество порций

- `ingredients` — массив объектов продукта:

    - `productId`

    - `quantity` (в базовых порциях)

    - `measure` (гр, мл, шт)

    - `price` (берется из таблицы products)

    - `totalPrice`

- `finalPrice` — общая стоимость (на текущие порции)


### 3.2. Продукт

- `id`

- `emoji` или `icon`

- `name`

- `unit` (шт, гр, мл — опционально, можно добавить)

- `price` — цена за единицу

- `defaultQuantity?` — можно добавить для удобства

## 4. **Требования к интерфейсу (UI)**

### 4.1. Общие

- Хедер на всех страницах со ссылками:

    - Recipes (главная)

    - Goods (продукты)

    - Log in / Profile

    - Settings

- Адаптивность (минимальный уровень — планшеты).


### 4.2. Главная страница (Recipes)

- Сетка карточек рецептов (как сетка видео YouTube).

- На каждой карточке:

    - название

    - фото

    - краткое описание

    - итоговая стоимость

- Панель фильтров:

    - по продуктам

    - по итоговой стоимости (min-max)

    - по алфавиту

- Поиск по названию рецепта.


### 4.3. Страница Products

- Список продуктов столбцом.

- У каждого продукта отображается:

    - смайлик/иконка

    - название

    - цена

    - единица меры

    - кнопки **edit** и **delete**

- Поиск по названию.

- Кнопка “Добавить продукт” внизу списка.


### 4.4. Страница рецепта (Детальная)

- Фото, название, описание.

- Настройка количества порций (инпут или +/–).

- Список ингредиентов:

    - смайлик

    - название

    - количество

    - единица меры

    - цена за единицу

    - стоимость ингредиента

- Итоговая стоимость рецепта.

- Кнопки:

    - "Редактировать рецепт"

    - "Удалить рецепт"


### 4.5. Страницы аутентификации

- Вход

- Регистрация

- Страница профиля (изменение имени, аватарки, выход)


### 4.6. 404

- Простая страница с кнопкой "На главную".

## 5. **Технические требования**

- Сборщик: **Vite**

- Фреймворк: **React**

- Язык: **TypeScript**

- Препроцессор стилей: **SCSS**

- Линтеры: **ESLint + Prettier**

- База данных: **Supabase + PostgreSQL**

- Библиотеки:  Zustand, TanStack Query

- Хостинг: Supabase (БД) + любой frontend хостинг (Vercel/Netlify)


---

## 6. **Аутентификация**

- Использовать встроенную авторизацию Supabase.

- Поддержка:

    - email/password

    - вход / выход

    - хранение профиля

- Защищённые роуты для:

    - создания рецептов

    - редактирования продуктов


---

## 7. **Роутинг**

- `/` — главная (рецепты)

- `/goods` — список продуктов

- `/recipe/:id` — детальная

- `/profile` — профиль пользователя

- `/auth` — логин/регистрация

- `/settings` — настройки

- `*` — 404
